# Анализ багов Timer Widget v1.2.9

## Средние

### BUG-01: e2e тест захардкожен под macOS
- **Файл**: `e2e/app.spec.js:10`
- **Проблема**: `executablePath` указывает на `Electron.app/Contents/MacOS/Electron` — тест не работает на Windows/Linux.
- **Риск**: Средний — e2e тесты невозможно запустить на Windows (основная платформа проекта).
- **Решение**: Убрать `executablePath`, использовать автоматическое определение пути через `require('electron')` или `electron.executablePath()`.
- **Статус**: [ ] Не исправлено

### ~~BUG-02: widget cleanup() может не вызываться~~
- **Файл**: `electron-widget.html:1708`
- **Статус**: [x] Проверено — `beforeunload` обработчик присутствует. Не баг.

### BUG-03: AudioContext race condition
- **Файл**: `electron-control.html` (SoundManager)
- **Проблема**: Быстрые последовательные вызовы `playPreset()` могут создать несколько AudioContext. Браузеры ограничивают количество активных контекстов.
- **Риск**: Средний — при быстром нажатии кнопок звука теоретически возможна ошибка.
- **Решение**: Кэшировать AudioContext и переиспользовать.
- **Статус**: [ ] Не исправлено

## Низкие

### BUG-04: Свойства showCurrentTime/showEventTime инициализируются, но не читаются
- **Файл**: `display-script.js:36-38`
- **Проблема**: `this.showCurrentTime`, `this.showEventTime` устанавливаются в конструкторе, но видимость блоков управляется напрямую через `applyDisplaySettings()` с CSS классами. Свойства не используются нигде кроме инициализации.
- **Риск**: Низкий — не баг, но мёртвое состояние, вводящее в заблуждение.
- **Решение**: Удалить неиспользуемые свойства.
- **Статус**: [ ] Не исправлено

### BUG-05: `timer-control` IPC обработчик принимает action как объект или строку
- **Файл**: `electron-main.js:692`
- **Проблема**: `timer-control` handler ожидает строку (`action`), но `display-script.js:83` отправляет `this.ipcRenderer.send('timer-control', 'pause')` через ipc-compat.js, который оборачивает данные. В ipc-compat `send()` просто проксирует — это ОК. Но в CLAUDE.md документирован формат `{ action: 'start' }`, а реальный код отправляет просто строку.
- **Риск**: Низкий — код работает, но документация в CLAUDE.md не соответствует реальности.
- **Решение**: Обновить CLAUDE.md — формат `timer-control` это строка, не объект.
- **Статус**: [ ] Не исправлено
